# 3.	Контекст

Проект разрабатывается в международной компании,
предназначен для самостоятельных тренировок, имеет интеграции с внешними приложениями по продажам,
т.е. возможен доступ до банковских данных, хранит персональные данные клиента,
в функционал приложения входит поиск по сохраненным данным. Возможны параллельные запросы на внесение
изменений в данные.

Т.о. База данных должна обеспечивать 
Мощный механизм транзакций и контроль целостности данных при любом числе пользователей.
Поддерживает обширный список встроенных типов данных, создание новых типов данных пользователем.
Возможность быстрого поиска (индексы).

Рассматриваем два варианта:
MySQL и PostgreSQL, PostgreSQL - работает на большинстве операционных систем из коробки,
платформа открытая и бесплатная, так как компания международная, а предположим, что сейчас частично 
разработчики находятся в том числе и в России, есть риски по закупке лицензий от MySQL.

# 4.   Решение

Мы решили использовать реляционную базу данных PostgreSQL, из-за высокой производительности,
с ней знакомы все члены команды разработки.

# 5.  Статус: Принято

# 6.  Последствия 

Мы начали разработку приложения, части, которая отчвечает за хранению информации, не тратя время

Минимизировали финансовые траты.

Так как в будущем приложение надеемся станет популярным к PostgreSQL сверху легко будет прикрутить
