# Нефункциональные требования



##  сервис ЛК клиента

1.  Система должна иметь двухфаторную авторизацию
2.  Использование протокола HTTPS для шифрования передаваемых данных
3.  Доступность системы должна быть 99% и выше
4.  Система должна гарантировать полное восстановление за 1,5 часа
5.  Критические бизнес сценарии должны работать при проблемах со связью
4.  Реализация должна быть выполнена таким образом, чтобы добавление новых возможностей в профиле занимало не более 1 спринта, фич не более квартала
6.  Должна быть встроена локализация для 8 международных языков
7.  Должен быть настроен мониторинг на подозрительные действия пользователей, при изменении "подозрительной" геопозиции, должно срабатывать оповещение клиенту на почту, осуществляться автоматический выход со всех устройств, запрашиваться двухфазная авторизация
10. Скорость обработки запросов для синхронизации не должна превышать 5 сек
11. При низкой пропускной способности сети картинки, видео не грузятся, либо подгружаются в низком качестве
12. Приложение должно поддерживать устройства, работающие на операционных системах: Android 14 и выше, iOS 16 И выше
13. Синхронизация данных ЛК на устройство клиента 1 раз в 5 часов

##  сервис Тренировки
1.  Приложение должно поддерживать устройства, работающие на операционных системах: Android 14 и выше, iOS 16 И выше
2.  Система должна иметь систему аутентификации и авторизации
3.  Вызов любого метода не должен превышать 1,5 с
4.  Должна быть встроена локализация для 8 международных языков
3.  При понижении пропускной способности сети, понижать качество ресурсов при отображении (скорость сети менее 1 мб)
4.  Доступность системы должна быть 99% и выше

##  сервис Реклама
1.  Система должна иметь систему аутентификации и авторизации, для определения кому какие доступны компоненты для управления
2.  Время срабатывания методов гет менее 5 сек
3.  Интерфейс администратора должен работать в Google Chrome
4.  Массовая загрузка промоакций поддерживает загрузку в формате json, файла до 300 мб, файлы вложенных картинок товаров каждый не должен превышать 16 мб
3.  Все интеграции с поставщиками промоакций и товаров должны быть обложены интеграционными тестами
4.  Мониторинг внутренних ошибок настроен на все 500 ошибки должно быть оповещение поддержки.
 

##  сервис Группы
1.  Система должна иметь систему аутентификации и авторизации(Ролевая модель должна быть реализована)
2. Должна быть настроена репликация данных раз в сутки
3. Должен быть настроен мониторинг действий пользователя на спам и прочее
4. Определено время восстановления
5. Скорость обмена сообщениями не более 1,5 сек
