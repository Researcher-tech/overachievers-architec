# Нефункциональные требования


К приложению в целом:

1. Система должна иметь систему аутентификации и авторизации
2. Приложение должно поддерживать устройства, работающие на операционных системах: Android 14 и выше, iOS 16 И выше
3. Должна быть встроена локализация
4. В зависимости от геоположения должны быть предусмотрены разные формы подписания соглашения об использовании приложением перс данных
5. Работа с критическими сценариями без связи с интернетом

##  сервис Клиенты

1.  Система должна иметь систему аутентификации и авторизации
2.  Система должна гарантировать восстановление за определенное время после критического сбоя
3.  Использование протокола HTTPS для шифрования передаваемых данных
3.  Система должна иметь защиту от DDos атак
4.  Реализация должна быть выполнена таким образом, чтобы интеграция и реализация нового функционала не занимала много времени
5.  Должно поддерживаться выполнение критичных сценариев без подключения к сети интернет
6.  Должна быть встроена локализация
7.  Должен быть настроен мониторинг на действия пользователя / на подозрительные действия пользователей
8.  Должна быть настроена система оповещения при сбоях
9.  Должна быть настроена репликация данных более частая чем для остальных модулей
10. Скорость обработки запросов для синхронизации не должна превышать 5 сек
11. При низкой пропускной способности сети картинки, видео не грузятся, либо подгружаются в низком качестве
12. Должен быть удобный интерфейс
13. Легкая тестируемость
14. Определено время отката к предыдущей версии



##  сервис Тренировки
1.  Система должна иметь систему аутентификации и авторизации
2.  Должен быть удобный интерфейс
3.  В зависимости от качества сети, понижать качество ресурсов при отображении
4.  Определено время ответа на методы получения тренировок, создания тренировок, удаления тренировок
6.  Определено время отката к предыдущей версии
7.  Настроена локализация
8.  Мониторинг внутренних ошибок


##  сервис Реклама
1.  Система должна иметь систему аутентификации и авторизации
2.  Определено время ответа на методы предоставления информации
3.  Все интеграции с поставщиками промоакций и товаров должны быть обложены интеграционными тестами
4.  Мониторинг внутренних ошибок
 


##  сервис Группы
1.  Система должна иметь систему аутентификации и авторизации
2.  Должна быть настроена репликация данных
3.  Должна быть настроен мониторинг действий пользователя (в том числе обнаружение подозрительных действий, спам и прочее)
4. Определено время восстановления
